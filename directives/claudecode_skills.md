# SOP: Implementação de Skills e Ultra Think
## Regras de Negócio Inegociáveis
1. Todo código gerado deve passar pelo comando 'simplify' antes de qualquer commit.
2. Tarefas repetitivas do SaaS (como scaffold de tabelas no Supabase ou criação de componentes UI padrão) devem ser transformadas em Skills usando o Skill Creator.
3. Modo 'Ultra Think' (raciocínio estendido) é obrigatório para: refatoração de banco de dados, debug de erros críticos de integração e arquitetura de novas features B2B.
